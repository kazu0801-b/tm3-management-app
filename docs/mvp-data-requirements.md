# MVP データ要件整理

## 目的

MVPでアプリを動かすために必要なデータを洗い出し、必要なテーブルと役割を明確にする。

この資料では、MVPで扱うテーブル一覧、各テーブルの役割、今回やらないDB設計を整理する。

---

## MVPで必要なテーブル一覧

| テーブル名 | 役割 |
|---|---|
| users | アプリを利用するユーザー情報を管理する。 |
| roles | ユーザーの権限や役割を管理する。 |
| projects | 管理対象となるプロジェクト情報を管理する。 |
| project_members | プロジェクトに参加しているユーザーと役割を管理する。 |
| tasks | プロジェクトに紐づくタスク情報を管理する。 |
| statuses | タスクの進捗状態を管理する。 |

---

## 各テーブルの説明

### users

アプリを利用するユーザー情報を管理するテーブル。

MVPでは、Directus標準のユーザー管理機能を使用するため、独自の users テーブルは作成しない方針とする。

主に担当者、作成者、プロジェクト参加者として利用する。

---

### roles

ユーザーの権限や役割を管理するテーブル。

MVPでは、Directus標準の権限機能を使用するため、独自の roles テーブルは作成しない方針とする。

権限はまず「管理者」「メンバー」の2種類を基本とする。

---

### projects

管理対象となるプロジェクト情報を管理するテーブル。

タスクは必ず1つのプロジェクトに紐づく想定とする。

プロジェクト名、説明、作成者、作成日時、更新日時などを持つ。

---

### project_members

プロジェクトに参加しているユーザーと、そのプロジェクト内での役割を管理するテーブル。

担当者一覧は、この project_members に登録されているユーザーを対象とする。

プロジェクト作成時には、作成者を project_members に登録する方針とする。

---

### tasks

プロジェクトに紐づくタスク情報を管理するテーブル。

タスク名、説明、担当者、ステータス、開始日、期限日、作成者などを持つ。

MVPでは、タスクの作成・一覧表示・編集・削除を行うための中心テーブルとする。

---

### statuses

タスクの進捗状態を管理するテーブル。

MVPでは以下の4種類を使用する。

| id | name | sort_order |
|---:|---|---:|
| 1 | 未対応 | 1 |
| 2 | 対応中 | 2 |
| 3 | 確認中 | 3 |
| 4 | 対応済み | 4 |

タスク作成時の初期ステータスは「未対応」とする。

カンバン画面では、この statuses をもとに列を表示する。

---

## MVPで必要な主なリレーション

| 関係 | 内容 |
|---|---|
| projects → tasks | 1つのプロジェクトに複数のタスクが紐づく。 |
| projects → project_members | 1つのプロジェクトに複数の参加者が紐づく。 |
| users → project_members | 1人のユーザーが複数プロジェクトに参加できる。 |
| users → tasks | 1人のユーザーが複数タスクを担当できる。 |
| statuses → tasks | 1つのステータスに複数タスクが紐づく。 |

---

## 今回やらないDB設計

MVPでは開発期間と実装コストを考慮し、以下のDB設計は対象外とする。

| 対象外 | 理由 |
|---|---|
| comments | コメント機能はMVPでは対象外のため。 |
| notifications | 通知機能は実装コストが高いため。 |
| attachments | ファイル添付はストレージ設計が必要になるため。 |
| activity_logs | 操作履歴はMVP後の拡張でよいため。 |
| task_dependencies | ガントの依存関係はMVPでは扱わないため。 |
| teams | 複数チーム管理は権限設計が複雑になるため。 |
| labels / tags | タスク分類はステータスと担当者で最低限対応するため。 |
| milestones | 期限管理は開始日・期限日で最低限対応するため。 |
| task_comments | タスク詳細内コメントはMVPでは対象外のため。 |
| task_histories | ステータス変更履歴はMVPでは対象外のため。 |

---

## MVPでの方針

MVPでは、以下の6つを基本データとして扱う。

- users
- roles
- projects
- project_members
- tasks
- statuses

ただし、users と roles は Directus標準機能を使用するため、独自テーブルとしては作成しない。

独自に管理する主なテーブルは以下とする。

- projects
- project_members
- tasks
- statuses
