# Directus 初期データ

## 目的

MVPでアプリを動かすために必要な初期データを整理する。

タスク作成画面やカンバン画面で使用するステータス、権限、テスト用データの方針を明確にする。

---

## statuses 初期データ

| id | name | sort_order |
|---:|---|---:|
| 1 | 未対応 | 1 |
| 2 | 対応中 | 2 |
| 3 | 確認中 | 3 |
| 4 | 対応済み | 4 |

---

## roles / 権限方針

MVPでは、roles テーブルを自作せず、Directus 標準の権限機能を使用する。

使用する権限は以下の2種類とする。

| 権限 | 説明 |
|---|---|
| 管理者 | プロジェクト作成、タスク管理、メンバー管理ができる |
| メンバー | プロジェクト参加、タスク作成・編集、ステータス変更ができる |

---

## users 方針

users テーブルは自作せず、Directus 標準のユーザー管理機能を使用する。

担当者として表示するユーザーは、project_members に登録されているユーザーを対象とする。

---

## project_members 方針

プロジェクト作成時、作成者を project_members に登録する。

担当者一覧は、プロジェクト参加者のみを対象とする。

---

## tasks 方針

タスク作成時の初期ステータスは「未対応」とする。

タスクには以下の情報を持たせる。

| 項目 | 内容 |
|---|---|
| project_id | 紐づくプロジェクト |
| title | タスク名 |
| description | タスク詳細 |
| status_id | ステータス |
| assignee_id | 担当者 |
| start_date | 開始日 |
| due_date | 期限日 |
| created_by | 作成者 |

---

## Directus上での確認内容

- statuses に4件登録されている
  - 未対応
  - 対応中
  - 確認中
  - 対応済み

- テスト用プロジェクトを1件登録できる
- テスト用ユーザーを2名用意できる
- project_members にプロジェクト参加者を登録できる
- tasks にテストタスクを1件登録できる
- tasks 作成時に project を選択できる
- tasks 作成時に status を選択できる
- tasks 作成時に assignee を選択できる
- tasks 一覧でステータス、担当者、開始日、期限日を確認できる

---

## 補足

Directus上の実データ登録はDB内の操作になるため、Gitには初期データの整理内容をドキュメントとして残す。
